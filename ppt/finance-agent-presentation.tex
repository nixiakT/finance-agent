\begin{frame}{问题：为什么不能直接问大模型“这只股票能买吗”？}
  \vspace{0.12cm}
  \begin{columns}[T,onlytextwidth]
    \column{0.31\textwidth}
      \begin{tikzpicture}
        \node[rounded corners=4pt,draw=zjured!55,fill=white,text width=3.9cm,

          {\color{zjured}\Large\bfseries 01}\quad{\bfseries 数据会过时}\par\vspace{0.28cm}
          {\small 行情、财报和新闻持续变化，模型参数中的知识不能代替实时查询。}\par\vspace{0.30cm}
          {\color{zjugray}\scriptsize 后果：引用旧价格或旧事件}
        };
      \end{tikzpicture}
    \column{0.31\textwidth}
      \begin{tikzpicture}
        \node[rounded corners=4pt,draw=zjuamber!65,fill=white,text width=3.9cm,
          minimum height=3.55cm,inner sep=0.27cm,align=left] {
          {\color{zjuamber}\Large\bfseries 02}\quad{\bfseries 事实难核验}\par\vspace{0.28cm}
          {\small 单一来源可能缺失或冲突；模型还可能补出看似合理但无依据的数字。}\par\vspace{0.30cm}
          {\color{zjugray}\scriptsize 后果：结论与证据脱节}
        };
      \end{tikzpicture}
    \column{0.31\textwidth}
      \begin{tikzpicture}
        \node[rounded corners=4pt,draw=zjublue!55,fill=white,text width=3.9cm,
          minimum height=3.55cm,inner sep=0.27cm,align=left] {
          {\color{zjublue}\Large\bfseries 03}\quad{\bfseries 观点容易单一}\par\vspace{0.28cm}
          {\small 同一模型常沿着最先形成的叙事继续推理，并给出过度确定的判断。}\par\vspace{0.30cm}
          {\color{zjugray}\scriptsize 后果：风险和反例被忽略}
        };
      \end{tikzpicture}
  \end{columns}
  \vspace{0.20cm}
  \begin{center}
    \colorbox{zjupale}{\parbox{0.91\textwidth}{\centering\small
      我们要解决的不是“让模型记住更多金融知识”，而是让它\textbf{查到、核对、引用并限制结论}。}}
  \end{center}
  \note{【35 秒】普通大模型直接做股票研究有三个问题。第一，参数里的金融知识很快过时；第二，真实数据源会缺字段、延迟甚至互相冲突，模型还可能补出没有依据的数字；第三，单个模型容易顺着一种叙事走到底。因此我们的目标不是让模型背更多股票知识，而是让它知道什么时候查工具，并让每个结论都能回到证据和质量检查。}
\end{frame}

% 4. Challenges and innovations ---------------------------------------------
\begin{frame}{挑战与创新：只解决三个真正影响结论的问题}
  \vspace{0.05cm}
  \begin{columns}[T,onlytextwidth]
    \column{0.32\textwidth}
      \begin{tikzpicture}
        \node[rounded corners=4pt,draw=zjublue!55,fill=white,text width=4.1cm,
          minimum height=4.45cm,inner sep=0.25cm,align=left] {
          {\color{zjublue}\bfseries 多源缺失与不一致}\par\vspace{0.18cm}
          {\small\bfseries 挑战}\quad 行情、历史和财务数据覆盖不同。\par\vspace{0.20cm}
          {\small\bfseries 做法}\quad ProviderChain 并发查询，按时效和完整度选择/融合，并保留 coverage 与 conflict。\par\vspace{0.26cm}
          \course{工具调用}\quad\course{并行执行}
        };
      \end{tikzpicture}
    \column{0.32\textwidth}
      \begin{tikzpicture}
        \node[rounded corners=4pt,draw=zjuamber!65,fill=white,text width=4.1cm,
          minimum height=4.45cm,inner sep=0.25cm,align=left] {
          {\color{zjuamber}\bfseries 单模型偏见与过度自信}\par\vspace{0.18cm}
          {\small\bfseries 挑战}\quad 同一模型容易忽略反例。\par\vspace{0.20cm}
          {\small\bfseries 做法}\quad 五个角色独立分析；Bull/Bear 交叉反驳；Judge 只基于同一证据集裁决。\par\vspace{0.26cm}
          \course{多智能体协作}\quad\course{LLM-as-Judge}
        };
      \end{tikzpicture}
    \column{0.32\textwidth}
      \begin{tikzpicture}
        \node[rounded corners=4pt,draw=zjured!55,fill=white,text width=4.1cm,
          minimum height=4.45cm,inner sep=0.25cm,align=left] {
          {\color{zjured}\bfseries 结论缺少证据与边界}\par\vspace{0.18cm}
          {\small\bfseries 挑战}\quad 有工具仍可能“引用错数字”。\par\vspace{0.20cm}
          {\small\bfseries 做法}\quad 编号证据、数字校验、QualityGate 评级；字段不足时删除说法并限制把握上限。\par\vspace{0.26cm}
          \course{上下文契约}\quad\course{安全机制}
        };
      \end{tikzpicture}
  \end{columns}
  \note{【45 秒】我们没有把普通功能都叫创新，而是集中解决三个影响结果可信度的问题。第一，多源数据缺失和冲突，所以 ProviderChain 不是简单拼接，而是选择、融合并记录覆盖情况。第二，单模型容易片面，所以让五个角色独立分析，再让 Bull 和 Bear 互相反驳，最后交给 Judge。第三，模型即使拿到工具结果也可能引用错数字，所以所有观点必须引用编号证据，代码再做数字校验和质量评级。}
\end{frame}

  \vspace{0.08cm}
  \begin{center}
    \small 相比单模型直接回答：\textbf{数据缺口被显式暴露，分歧被保留，无依据数字被程序拦截。}
  \end{center}
  \note{【75 秒】我们用 AAPL 做了一次关闭样例回退的真实运行。系统从 Yahoo 得到 317.31 美元行情和 251 个历史点，从 AKShare 得到财务数据，并计算 MA、RSI 和波动率。数据看起来很多，但 QualityGate 只给 B：PE 和自由现金流缺失，而且行情、历史和财务实际上都缺少充分的跨源验证。Bull 看到了盈利和动量，Bear 强调高杠杆、缺失字段和近期涨幅风险。Judge 最终给出 4 分、30 天 down，模型原本请求 55\% 的把握，但代码因为数据覆盖和校验问题把上限压到 40\%。这里的 40\% 不是统计校准概率，只是受约束的模型信号。更重要的是，无证据数字被删除，无法验证的结论被替换为需人工复核。}
\end{frame}

% 7. Reflection ---------------------------------------------------------------
\begin{frame}{思考：多一个 Agent，不等于多一份真相}
  \vspace{0.10cm}
  \begin{columns}[T,onlytextwidth]
    \column{0.57\textwidth}
      \begin{enumerate}\small
        \item \textbf{多智能体是否更准确？}\quad 角色增加了反例，但也增加成本和共同错误的机会。
        \item \textbf{Judge 是否真正独立？}\quad 它仍可能受前序角色的表达顺序和框架影响。
        \item \textbf{数据是否足够新、足够全？}\quad AAPL 案例已暴露单源覆盖与关键字段缺失。
        \item \textbf{回测是否能代表未来？}\quad 只能检验历史流程，不能保证未来收益。
      \end{enumerate}
    \column{0.39\textwidth}
      \begin{tikzpicture}
        \node[rounded corners=5pt,draw=zjured!60,fill=zjured!6,text width=4.75cm,
          minimum height=3.75cm,inner sep=0.30cm,align=center] {
          {\color{zjured}\Large\bfseries 边界}\par\vspace{0.30cm}
          {\bfseries 研究辅助，不是交易系统}\par\vspace{0.20cm}
          {\small 不执行真实交易\par
          不把回测当收益承诺\par
          信息不足时允许“不下结论”}\par\vspace{0.26cm}
          \course{安全机制}\quad\course{可观测性}
        };
      \end{tikzpicture}
  \end{columns}
  \note{【30 秒】这个项目仍有明显限制。多智能体不一定比单模型更准确，它可能只是让分歧更可见；Judge 也可能受到前面角色的表达框架影响。金融数据的时效和覆盖还受外部 API 限制，回测更不能代表未来收益。所以我们把系统定位为研究辅助，并允许信息不足时不下结论，也不执行真实交易。}
\end{frame}
